# SIGILFAE PYXIS

> Upload-box fairies. Real file-form behaviour. Token: `,`

A lavender-bodied fairy with a small white-outlined upload-box icon on her
chest in place of the pentagon. The box has an up-arrow inside it.

## Behaviour

PYXIS fairies carry **real file-upload forms**. Clicking the upload-box on
her chest opens the device's native file picker. Selecting a file:

- Attaches it to the fairy (state: `has-file`)
- Auto-progresses the node from `open` → `in-progress`
- Filename appears in the state tag below the fairy
- Box icon turns green

Dragging the fairy still works as in NOTAE.

## Why "Pyxis"?

Latin for "small box". Originally a small chalice-or-container; the empire
borrows the word for a fairy who carries a box. Etymologically clean,
phonetically distinct.

## DOTGRAMMA usage

```
,           → 1 PYXIS fairy at centre
,-,         → 2 PYXIS fairies on a line, edge between
,-,-,-,-    → 4 PYXIS fairies in closed square (4 file-upload forms)
.-,-.-,-    → mixed NOTAE + PYXIS, alternating
```

## Connection to mcFORM

PYXIS is the bridge between DOTGRAMMA and the empire's existing mcFORM tool.
Where mcFORM is the form-builder, PYXIS is the form-renderer-as-character —
forms become embodied, draggable, popable.
